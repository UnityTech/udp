Brief Demo#

Editor Script#

Editor script can be used to onboard client and register user.

  1. First, open the Unity targeting our INT/STAGING environment (Mac OS for example):

  2. Login with an INT/STAGING account which can be registered at our INT ID portal or STAGING ID Portal.

  3. Create a new Project (old project has to unlink the project id and generate a new one) and import the package.
  4. Create an Game Settings
  5. Generate an Unity client and create test account.
  6. Open SampleScene
  7. Open Window -> Unity UDP -> IAP Catalog
  8. Add Product Draft and click Create Product to sync with the server. Price must be set in order to purchase.
  9. Click the GameManager GameObject in the Hierarchy and fill the slug into Product_1 and Product_2. More products can also be added through GameManager.cs file.
  10. Build and open the App
  11. Init and login are bonded together and for UDP, developer must login using the test account.
  12. After login is succeeded, click Buy or Buy&Consume to buy the product registered. Purchase information will be shown.
  13. If the purchase is not consumed, it cannot be purchased again.
  14. Click QueryInventory or Query&Consume, all the products and unconsumed purchases will be returned.

Alpha v0.1#

Brief Demo#

Editor Script#

Editor script can be used to onboard client and register user.

  1. First, open the Unity targeting our INT env (Mac OS for example):

  2. Login with an INT account which can be registered at our INT ID portal.

  3. Create a new Project (old project has to unlink the project id and generate a new one) and import the package.
  4. Create an AppStoreSettings
  5. Generate an Unity client and create test account.
  6. Save all setting.
  7. Open SampleScene
  8. Open Window -> Unity UDP -> IAP Catalog
  9. Add Product and click the plus sign on the upper-right (next to the x symbol) to sync with the server and get the ProductId.
  10. Open Scripts/GameManager.cs and fill the product id in the variant SKU.
  11. Build and open the App
  12. Init and login are bonded together and for UDP, developer must login using the test account.
  13. After login is succeeded, click Purchase to buy the product registered. (This flow will be optimized to avoid hard coding the product id)
  14. If the purchase is not consumed, it cannot be purchased again.
  15. Click queryInventory, all the products and unconsumed purchases will be returned. In the demo, we consume the purchase here.
