Get Started

Unity Distribution Portal (UDP) is a IAP-focused product from Unity which lets you distribute a single game build to multiple Android app stores. UDP repacks your Android build with each store’s dedicated IAP SDK, and lets you manage all your submissions on the UDP portal. UDP handles all the heavy lifting and provides you with a streamlined distribution flow by integrating with the app stores. UDP is currently in private Beta.

To get started, download and install either of the two packages:

  • UDP Package for Unity IAP
    If you’re already using Unity IAP in your game, we recommend you use the Unity IAP Package. Alternatively, update your Unity IAP asset store package to version 1.21 which readily includes UDP.
  • Standalone SDK
    If you are not currently using Unity IAP, we recommend you use the stand-alone UDP SDK. Its implementation is similar to that of mainstream IAP SDKs, and slightly more direct.

Refer to the UDP documentation for implementation details. Regardless of which SDK you decide to use, follow these steps:

  1. In the Unity Editor, implement UDP in your game
    Create a new UDP game instance via the UDP Game Settings window, populate your IAP Catalog, and map your IAPs to your actual in-game items.

  2. Build and deploy your game to UDP
    Build your UDP package locally or via CloudBuild, and deploy it to the UDP portal to prepare for distribution.

  3. Prepare your game for distribution
    On the UDP portal, provide the information and resources for your game, such as multi-language descriptions, screenshots, and video trailers, set up your IAPs and create releases for distribution.

    Note: Beta access is currently restricted, if you have issues accessing the UDP portal, please contact udpsupport@unity3d.com.

  1. Select the stores that you want to distribute your game to
    Register your game with the app stores directly from inside the UDP portal. Before you submit to these stores, you can get your QA team to test the final builds, and push your game to your beta users.

  2. Publish
    Select the stores you want to publish your game to and submit it to the stores. Once your game is published, monitor its performance in the various stores from the UDP reporting dashboard.

Refer to the UDP documentation for implementation details. If you have any questions or need help to gain access to UDP, please write to udpsupport@unity3d.com.
